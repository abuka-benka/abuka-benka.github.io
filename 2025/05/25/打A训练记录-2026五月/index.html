CF2230F Game on Growing Tree

先考虑游戏进程，发现是A选一个点，然后B堵死一个最优邻居，然后A移向一个次优邻居。

考虑可以发现对于n个节点，最大化答案的树形态就是二叉树。因此答案是log n 的，本题不超过18。

因此二分，把答案序列二分成18份，check是O(n)的。

如何check?

维护dp值是以x子树为一个game，从x出发的最大游戏轮数。那么dp[x]就是第二大的dp[y]+1。然后换根，换根时需要维护前三大，有点麻烦。

法一：

一眼树上背包，考虑最小联通子图，肯定是一个三菱车标类似物 或 退化为一条链，但我们都可以枚举中间那个节点，然后换根 dp ，枚举每个点作为根节点时，选三个子树的链，总长和为 ，可以看出就是选三个多项式相乘，取 的系数。然后，NTT 就可以 草过去。

法二：

显然法一不是啥好做法。我们可以考虑优化下。考虑改成枚举 x 的子树有1，2个节点长度为 k 的答案。这样每两个节点只会在 lca 节点处理一次，复杂度就是 了。

CF2231F

拉格朗日四平方定理：每个正整数都可以表示为至多 4 个正整数的平方和

所以答案至多为 4 。

只需要 check 答案是否是 1，2，3 。

1：很简单，判断一个数是不是平方数。

2：平方数是 量级的，那我们可以预处理一手，复杂度 O(n)，答案直接 O(1) 查询。

3：既然答案为 2 可以 O(1) 查询，那么我们可以 枚举第一步，然后 check 两步能不能到。
